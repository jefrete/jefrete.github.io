Introducción

HTML (HyperText Markup Language) es el lenguaje estándar para crear páginas web. Utiliza etiquetas para estructurar el contenido en un documento que puede ser interpretado por los navegadores.



Estructura Básica de un Documento HTML

<!DOCTYPE html>
<html lang="es">
<head>
    <meta charset="UTF-8">
    <title>Título de la Página</title>
</head>
<body>
    <!-- Contenido va aquí -->
</body>
</html>

Para crear la estructura básica de un documento HTML5 utilizando atajos en los editores de código:

  • Con Emmet en Visual Studio Code o Sublime Text: Escribe html:5 y presiona Tab.
  • Con el atajo !: Escribe ! y presiona Tab.


Comentarios en HTML

Los comentarios en HTML se utilizan para dejar notas o explicaciones en el código que no serán visibles en el navegador. Son útiles para documentar el código y facilitar su comprensión por otros desarrolladores. 


<!-- Este es un comentario en HTML -->

que abarca múltiples
líneas.
-->

Etiquetas Básicas de HTML

Encabezados

Las etiquetas de encabezado definen niveles de importancia:

<h1>Encabezado 1</h1>
<h2>Encabezado 2</h2>

Párrafos

<p>Este es un párrafo.</p>

Enlaces

<a href="https://www.ejemplo.com">Visita Ejemplo</a>

Imágenes

<img src="ruta/a/imagen.jpg" alt="Descripción de la imagen">

Listas

<ul>
    <li>Elemento 1</li>
    <li>Elemento 2</li>
</ul>

<ol>
    <li>Elemento 1</li>
    <li>Elemento 2</li>
</ol>

Divisiones y Span

<div>
    <h2>Encabezado dentro de un div</h2>
    <p>Párrafo dentro de un div</p>
</div>

<p>
Texto con <span style="color:red;">
span rojo</span> incluido.
</p>

Formularios

<form action="/enviar" method="post">
    <label for="nombre">Nombre:</label>
    <input type="text" id="nombre" name="nombre">
    <input type="submit" value="Enviar">
</form>

Tablas

<table>
    <tr>
        <th>Encabezado 1</th>
        <th>Encabezado 2</th>
    </tr>
    <tr>
        <td>Celda 1</td>
        <td>Celda 2</td>
    </tr>
</table>

Etiquetas Semánticas

<header>
    <h1>Mi Sitio Web</h1>
</header>

<nav>
    <ul>
        <li><a href="inicio">Inicio</a></li>
        <li><a href="acerca">Acerca de</a></li>
        <li><a href="contacto">Contacto</a></li>
    </ul>
</nav>

<section>
    <h2>Sección Principal</h2>
    <p>Contenido de la sección principal.</p>
</section>

<aside>
    <h3>Información Adicional</h3>
    <p>Contenido adicional.</p>
</aside>

<footer>
    <p>© 2024 Mi Sitio Web</p>
</footer>

Buenas Prácticas

  • Usa etiquetas semánticas para mejorar accesibilidad y SEO.
  • Cierra todas las etiquetas, incluso si no es obligatorio.
  • Mantén el código limpio y ordenado.
  • Usa comentarios para documentar secciones del código.
  • Valida tu HTML usando validadores como W3C Markup Validation Service.

Malas Prácticas

  • No uses etiquetas no semánticas cuando hay disponibles otras que sí lo son.
  • No dejes etiquetas sin cerrar, pueden causar problemas de renderización.
  • Evita usar estilos en línea, utiliza CSS externo o interno.
  • Usa nombres descriptivos para clases e IDs.
